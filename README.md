# HomeMind AI

**Local AI security monitoring for Home Assistant — powered by Ollama, zero cloud required.**

HomeMind AI watches your cameras at night, uses a local vision model (Ollama/LLaVA) to evaluate what's happening, and sends you a concise morning report via Telegram. High-priority events (e.g. a person detected) trigger an immediate alert with a snapshot.

---

## Features

- **Night monitoring** — configurable window (e.g. 22:00 → 06:00)
- **AI vision analysis** — local Ollama model classifies events (person, vehicle, animal, suspicious activity…)
- **Smart filtering** — only relevant events are stored and reported, saving tokens and reducing noise
- **Telegram alerts** — immediate push for high-priority events + morning report with snapshot
- **Morning report** — daily summary at your chosen time, generated by AI in Italian
- **Snapshot storage** — images saved to `/config/www/homemind_snapshots/` and accessible via HA
- **Fully configurable from the HA UI** — no YAML editing needed
- **HACS compatible**

## Requirements

| Component | Details |
|-----------|---------|
| Home Assistant | 2024.1.0 or newer |
| [Ollama](https://ollama.ai) | Running locally on your server/NAS |
| Ollama model | `llava`, `llava-phi3`, or `moondream` (vision-capable) |
| Telegram Bot | Optional — for notifications |
| HA Camera entities | At least one camera integrated in HA |

### Install a vision model in Ollama

```bash
# Recommended (fast, good quality)
ollama pull llava-phi3

# Alternative (smaller, faster)
ollama pull moondream

# Full quality (needs more RAM)
ollama pull llava:13b
```

## Installation via HACS

1. Open HACS in your Home Assistant
2. Go to **Integrations** → click the three-dot menu → **Custom repositories**
3. Add `https://github.com/francescogalli-design/homemind-ai` as an **Integration**
4. Search for **HomeMind AI** and click **Download**
5. Restart Home Assistant
6. Go to **Settings → Devices & Services → Add Integration** → search **HomeMind AI**

## Configuration

The setup wizard walks you through 4 steps:

### Step 1 — Ollama AI
- **Ollama URL**: where Ollama is running (default: `http://localhost:11434`)
- **Vision Model**: the model to use for image analysis (default: `llava`)

### Step 2 — Telegram (optional)
- **Bot Token**: from [@BotFather](https://t.me/BotFather)
- **Chat ID**: your personal Telegram Chat ID (send `/start` to [@userinfobot](https://t.me/userinfobot))

### Step 3 — Cameras & Motion Sensors
- Select the **camera entities** to use for snapshots
- Select **motion binary sensors** that trigger analysis (optional — cameras can trigger directly)

> **Tip:** HomeMind AI auto-matches sensors to cameras by name prefix.
> `binary_sensor.front_yard_motion` → `camera.front_yard`

### Step 4 — Schedule
- **Night Start** (default: 22:00) — when monitoring activates
- **Night End** (default: 06:00) — when monitoring deactivates
- **Morning Report Time** (default: 07:00) — when the daily summary is sent

All settings can be changed later via **Settings → Devices & Services → HomeMind AI → Configure**.

## Example Automation

```yaml
# Send an immediate notification when HomeMind detects a high-priority event
automation:
  alias: "HomeMind — High Priority Alert"
  trigger:
    platform: event
    event_type: homemind_ai_alert
    event_data:
      priority: high
  action:
    - service: notify.mobile_app_your_phone
      data:
        title: "🚨 Allerta Sicurezza"
        message: "{{ trigger.event.data.description }}"
        data:
          image: "{{ trigger.event.data.snapshot_url }}"
```

## Privacy

All processing is **100% local**. Images are analyzed by Ollama running on your own hardware. Nothing is sent to external servers (unless you configure Telegram notifications, which sends only the alert text and snapshot to Telegram's servers).

## Troubleshooting

**`sensor.homemind_ai_status` shows `ollama_offline`**
→ Verify Ollama is running: `curl http://localhost:11434/api/tags`
→ Check the Ollama URL in the integration settings.

**No alerts at night even with motion detected**
→ Make sure your motion sensors change state to `on` when triggered.
→ Verify the night window covers the current time.
→ Check HA logs for `homemind_ai` debug messages.

**Telegram not receiving messages**
→ Verify the bot token with: `curl https://api.telegram.org/bot<TOKEN>/getMe`
→ Make sure you've sent at least one message to your bot first.
